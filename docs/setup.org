#+OPTIONS: toc:2 num:nil

* Prerequisites Checklist

Before starting, verify you have the following:

** Python 3.10+

#+BEGIN_SRC bash
  python --version        # should print Python 3.10.x or higher
  # If not installed:
  # Ubuntu/Debian:  sudo apt install python3.11 python3.11-venv
  # macOS:          brew install python@3.11
  # Windows:        Download from https://python.org
#+END_SRC

** Git

#+BEGIN_SRC bash
  git --version
  # Ubuntu: sudo apt install git
  # macOS:  brew install git
#+END_SRC

** Ollama

Download from https://ollama.com/download

#+BEGIN_SRC bash
  # Verify after installing:
  ollama --version
#+END_SRC

* Step-by-Step Setup

** Step 1: Clone and enter the repository

#+BEGIN_SRC bash
  git clone https://github.com/<your-username>/code-vuln-scanner.git
  cd code-vuln-scanner
#+END_SRC

** Step 2: Create a Python virtual environment

A virtual environment keeps project dependencies isolated from your system Python.

#+BEGIN_SRC bash
  python -m venv .venv

  # Activate it:
  # Linux / macOS:
  source .venv/bin/activate

  # Windows (PowerShell):
  # .venv\Scripts\Activate.ps1

  # Windows (cmd.exe):
  # .venv\Scripts\activate.bat

  # Verify activation (should show .venv path):
  which python
#+END_SRC

** Step 3: Install Python dependencies

#+BEGIN_SRC bash
  pip install --upgrade pip
  pip install -r requirements.txt
  pip install -e .              # installs the package in editable mode
  # enables: vuln-scan <file> from any directory
#+END_SRC

Installing sentence-transformers and chromadb will download ~400 MB of dependencies on first run. This is normal.

** Step 4: Configure environment variables

#+BEGIN_SRC bash
  cp .env.example .env
#+END_SRC

Open =.env= in Emacs and set your values:

#+BEGIN_SRC bash
  emacs .env
#+END_SRC

Minimum required settings (the defaults work fine if you have 8 GB RAM):

#+BEGIN_EXAMPLE
OLLAMA_MODEL=codellama:7b
SKIP_LLM=false
#+END_EXAMPLE

Optional but recommended:

#+BEGIN_EXAMPLE
NVD_API_KEY=<your_key>     # get free at nvd.nist.gov/developers/request-an-api-key
#+END_EXAMPLE

** Step 5: Start Ollama and pull a model

Open a terminal and start the Ollama server. Keep this terminal open throughout your session.

#+BEGIN_SRC bash
  ollama serve
#+END_SRC

In a *second* terminal, pull the model:

#+BEGIN_SRC bash
  ollama pull codellama:7b          # ~3.8 GB download — good for 8 GB RAM machines
#+END_SRC

Alternative models (pick only one to start):

#+BEGIN_SRC bash
  ollama pull deepseek-coder:6.7b  # similar size, often better reasoning
  ollama pull codellama:13b        # best quality, needs 16 GB RAM
#+END_SRC

Verify the model is available:

#+BEGIN_SRC bash
  ollama list
#+END_SRC

** Step 6: Build the knowledge base

This step downloads vulnerability data and builds the ChromaDB vector index.

#+BEGIN_SRC bash
  python scripts/setup_db.py
#+END_SRC

Expected output includes:
- CWE XML downloaded to =data/raw/cwec_latest.xml= (~12 MB download)
- CVEs fetched from NVD API and cached to =data/cve_cache/=
- ChromaDB populated at =data/chromadb/=

This takes:
- Without NVD API key: 10–15 minutes
- With NVD API key: 1–2 minutes

You only need to run this once. Cached data persists across runs.

** Step 7: Verify everything works

#+BEGIN_SRC bash
  # Quick test — scan the SQL injection fixture (no LLM needed)
  vuln-scan tests/fixtures/vulnerable_code/sql_injection.py --no-llm

  # Full scan with LLM (Ollama must be running)
  vuln-scan tests/fixtures/vulnerable_code/insecure_deser.py

  # Run the unit test suite
  pytest tests/unit/ -v
#+END_SRC

** Step 8: Launch the web UI

#+BEGIN_SRC bash
  streamlit run src/vuln_scanner/ui/app.py
#+END_SRC

Open http://localhost:8501 in your browser.

* Troubleshooting

** "ollama: command not found"
Ollama is not installed or not in PATH. Reinstall from https://ollama.com.

** "Cannot connect to Ollama"
The server isn't running. Open a terminal and run: =ollama serve=

** "Model 'codellama:7b' not found"
The model hasn't been pulled. Run: =ollama pull codellama:7b=

** "CVE collection is empty"
The knowledge base hasn't been populated. Run: =python scripts/setup_db.py=

** Scan is very slow (>5 min)
Normal for 13B models on CPU. Switch to a smaller model:

#+BEGIN_SRC bash
  OLLAMA_MODEL=codellama:7b vuln-scan yourfile.py
#+END_SRC

** "sentence-transformers" takes too long to load
The embedding model is downloaded on first use (~90 MB for all-MiniLM-L6-v2).
Subsequent runs are fast because the model is cached.

** ChromaDB version conflict errors

#+BEGIN_SRC bash
  pip install chromadb==0.5.23 --force-reinstall
#+END_SRC

** ImportError for pydantic_settings

#+BEGIN_SRC bash
  pip install pydantic-settings==2.7.0
#+END_SRC
