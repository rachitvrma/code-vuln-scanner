* Overview

A production-grade security tool that combines *static analysis*, *Retrieval-Augmented Generation (RAG)*, and a *locally-running LLM* (via Ollama) to detect security vulnerabilities in source code.

Unlike traditional scanners that rely on fixed rule sets, this tool uses a language model to *reason* about code — the same approach taken by GitHub Copilot Autofix and Semgrep AI.

* Requirements

** Hardware
- RAM: 8 GB minimum (for 7B parameter models); 16 GB recommended
- Disk: ~5 GB for models, ~200 MB for the CWE database

** Software
- Python 3.10 or higher
- [[https://ollama.com][Ollama]] (local LLM server)
- Git

* Installation

** 1. Clone the repository

#+BEGIN_SRC bash
  git clone https://github.com/rachitvrma/code-vuln-scanner.git
  cd code-vuln-scanner
#+END_SRC

** 2. Create a virtual environment

#+BEGIN_SRC bash
  python -m venv .venv
  source .venv/bin/activate        # Linux / macOS
  # .venv\Scripts\activate         # Windows PowerShell
#+END_SRC

** 3. Install Python dependencies

#+BEGIN_SRC bash
  pip install -r requirements.txt
  pip install -e .                  # installs the vuln-scan CLI command
#+END_SRC

** 4. Install Ollama and pull a model

Visit https://ollama.com and install Ollama for your OS, then:

#+BEGIN_SRC bash
  ollama serve                      # start the server (keep this running)

  # In a separate terminal, pull a model (choose one):
  ollama pull codellama:7b          # recommended — fast, code-focused (~4 GB)
  ollama pull deepseek-coder:6.7b   # alternative — strong code reasoning (~4 GB)
  ollama pull codellama:13b         # best accuracy, needs ~16 GB RAM
#+END_SRC

** 5. Configure environment

#+BEGIN_SRC bash
  cp .env.example .env
  # Edit .env with your settings (model name, optional NVD API key, etc.)
#+END_SRC

** 6. Populate the vulnerability knowledge base

#+BEGIN_SRC bash
  python scripts/setup_db.py
#+END_SRC

This downloads the MITRE CWE database and fetches CVE records from NVD.
Runtime: ~10 min without an NVD API key, ~1–2 min with one.

To get a free NVD API key: https://nvd.nist.gov/developers/request-an-api-key

* Usage

** Web Interface (Streamlit)

#+BEGIN_SRC bash
  streamlit run src/vuln_scanner/ui/app.py
#+END_SRC

Open http://localhost:8501 in your browser.

** Command Line

#+BEGIN_SRC bash
  # Basic scan (auto-detects language)
  vuln-scan path/to/file.py

  # Specify language explicitly
  vuln-scan path/to/main.c --language c

  # Output as JSON
  vuln-scan app.py --format json --output report.json

  # Fast scan — pattern + static only (no Ollama needed)
  vuln-scan app.py --no-llm

  # Use a different model
  vuln-scan app.py --model deepseek-coder:6.7b
#+END_SRC

Exit codes: =0= no issues · =1= low · =2= medium · =3= high · =4= critical

** Evaluation

#+BEGIN_SRC bash
  python scripts/evaluate.py                 # run on all labelled fixtures
  python scripts/evaluate.py --no-llm       # pattern-only (fast)
  python scripts/evaluate.py --output eval_results.json
#+END_SRC

** Refresh CVE data

#+BEGIN_SRC bash
  python scripts/fetch_cve_data.py           # refresh all NVD keywords
  python scripts/fetch_cve_data.py --fresh   # ignore cache, re-download
#+END_SRC

** Concepts to study

- *RAG (Retrieval-Augmented Generation)* — the core technique used to ground LLM outputs in real CVE/CWE data
- *Vector embeddings* — how text is converted to numbers for similarity search
- *Cosine similarity* — how ChromaDB compares embeddings
- *Prompt engineering* — how the format/content of the LLM prompt affects output quality
- *CWE / CVE* — Common Weakness Enumeration and Common Vulnerabilities and Exposures
- *CVSS scoring* — how vulnerability severity is standardised
